# 🚕 Ride-Hailing Churn Intelligence System (Multi-Agent AI Project)

## 📌 Overview
This project is an **industry-level churn analysis system** built for ride-hailing platforms like Uber, Ola, and Rapido.

It combines:
- 📊 Machine Learning for churn prediction  
- 🤖 Multi-Agent Architecture for intelligent decision-making  
- 📈 Data Visualization for business insights  

👉 The system analyzes customer reviews and ride data to identify **churn risk** and generate **actionable strategies**.

---

## 🎯 Problem Statement
Ride-hailing platforms face major challenges:
- High customer churn due to poor experience
- Driver cancellations and long wait times
- App crashes and pricing dissatisfaction

❗ Existing systems lack **real-time intelligence** and **coordinated decision-making**

👉 This project solves it using a **Multi-Agent AI System**

---

## 🧠 Solution (Agent-Based Architecture)

The system is designed using multiple intelligent agents:

- 📱 **Data Collection Agent** → Collects multi-platform data  
- 🔄 **Data Processing Agent** → Standardizes datasets  
- 🧹 **Data Cleaning Agent** → Cleans and prepares data  
- 🤖 **ML Prediction Agent** → Predicts churn probability  
- 📊 **Churn Analysis Agent** → Identifies patterns  
- 🎯 **Top Reasons Agent** → Extracts churn causes  
- 💡 **Insights Engine** → Generates insights  
- 🚀 **Recommendation Agent** → Suggests retention strategies  

---

## 📂 Dataset Used

- Uber Reviews Dataset (~12,000 records)
- Ola Booking Dataset (~50,000 rides)
- SubRide Dataset (Reviews + Labels)
- Annotator Dataset (Human-labeled sentiment)

## 🤖 Machine Learning Models

- Random Forest  
- Logistic Regression  
- Gradient Boosting  
- Linear SVM ✅ (Best Performing Model)

---

## 📊 Results

- 🚀 Best Model: **Linear SVM**
- 📈 Accuracy: **92.72%**
- 🎯 F1 Score: **76.92%**
- ⚠️ Churn Rate: **9.1%**

---

## 🔑 Key Insights

Top churn indicators:
- Pricing dissatisfaction  
- Driver cancellations  
- App crashes  
- Long wait times
